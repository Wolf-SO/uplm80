uplm80 Compiler Bug Report
===========================

Bugs discovered while building 80un.plm (CP/M archive unpacker)

All of these appear to be register allocation bugs where uplm80 incorrectly
reuses registers or stack locations across procedure calls.


BUG 1: DOUBLE() builtin doesn't work correctly with SHL
--------------------------------------------------------
Code:
    DECLARE B BYTE;
    DECLARE RESULT ADDRESS;
    B = GETBYTE;
    RESULT = SHL(DOUBLE(B), 8);

Expected: High byte of RESULT contains B, low byte is 0
Actual: High byte contains garbage, low byte is 0

Workaround: Assign BYTE to ADDRESS variable first:
    DECLARE B BYTE;
    DECLARE TEMP ADDRESS;
    DECLARE RESULT ADDRESS;
    B = GETBYTE;
    TEMP = B;
    RESULT = SHL(TEMP, 8);


BUG 2: Local BYTE variables corrupted after CALL statements
------------------------------------------------------------
Code:
    DECLARE SIGLEVEL BYTE;
    SIGLEVEL = GETBYTE;        /* Reads 0x20 correctly */
    CALL PRINTSTR(.('test$'));
    /* SIGLEVEL now contains 0x0E instead of 0x20 */

Expected: Local BYTE variables retain their values across procedure calls
Actual: Values change to seemingly random values (possibly uninitialized
        register contents)

Workaround: Declare as ADDRESS instead of BYTE:
    DECLARE SIGLEVEL ADDRESS;


BUG 3: SHR/SHL broken for shift counts > 8
------------------------------------------
Code:
    DECLARE VAL ADDRESS;
    VAL = 02419H;
    VAL = SHR(VAL, 7);

Expected: VAL = 0x0048 (0x2419 >> 7 = 72)
Actual: VAL = 0x0000

Also affects SHL:
    SHL(1, 9) - 1 returns 0x0001 instead of 0x01FF

Workaround: Implement safe shift using multiplication/division:
    SHR$SAFE: PROCEDURE(VAL, CNT) ADDRESS;
        DECLARE VAL ADDRESS;
        DECLARE CNT BYTE;
        DO WHILE CNT > 0;
            VAL = VAL / 2;
            CNT = CNT - 1;
        END;
        RETURN VAL;
    END SHR$SAFE;

    SHL$SAFE: PROCEDURE(VAL, CNT) ADDRESS;
        DECLARE VAL ADDRESS;
        DECLARE CNT BYTE;
        DO WHILE CNT > 0;
            VAL = VAL * 2;
            CNT = CNT - 1;
        END;
        RETURN VAL;
    END SHL$SAFE;


BUG 4: Loop counter variables corrupted during iteration
---------------------------------------------------------
Code:
    DECLARE I BYTE;
    I = 0;
    DO WHILE I < 13;
        FILENAME(I) = GETBYTE;
        CALL PRINTHEX(FILENAME(I));
        I = I + 1;
    END;

Expected: Loop iterates 13 times
Actual: Loop exits after 1 iteration (I gets corrupted to >= 13)

This happens even when I is declared as ADDRESS.

Workaround: Unroll the loop:
    FILENAME(0) = GETBYTE;
    FILENAME(1) = GETBYTE;
    FILENAME(2) = GETBYTE;
    /* ... etc ... */


Test case: ~/src/80un/80un.plm
The working code includes all workarounds and can be used as a reference.
